
# PyRTL to LTspice Demo

This is a small proof-of-concept for generating LTspice `.cir` simulation files from simple PyRTL combinational circuits.

The current demo reads the circuit structure PyRTL builds and emits an LTspice simulation file. The generated file includes the main pieces needed to run a simulation: circuit instances, named wire connections, input voltage sources, power and ground, transistor/gate definitions, and a transient analysis command.

I first tested the idea on a half adder, then extended it to a small reusable PyRTL component library. The 4-bit ripple-carry adder example imports one reusable `full_adder()` module and composes four instances by connecting each carry output to the next carry input.

## Files

- `pyrtl_to_ltspice.py`  
  Minimal converter from PyRTL's working block to an LTspice `.cir` file.

- `ltspice_pyrtl_lib.py`  
  Small reusable PyRTL component library. Currently includes a `full_adder()` module.

- `examples/demo_half_adder.py`  
  Simple half-adder test circuit.

- `examples/demo_4bit_adder.py`  
  4-bit ripple-carry adder built from four imported full-adder modules.

- `generated/`  
  Example `.cir` files generated by the converter.

- `screenshots/`  
  LTspice waveform screenshots from the generated simulations.

## 4-bit adder composition

The 4-bit adder is built by importing and reusing one `full_adder()` function four times:

s0, c1 = full_adder(A0, B0, Cin, name="fa0")

s3, c4 = full_adder(A3, B3, c3,  name="fa3")
